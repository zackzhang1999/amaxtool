# AMAX Tool 优化改进报告

## 优化概述

本次优化对 AMAX Tool 进行了全面重构，解决了代码重复、风格不一致、缺乏错误处理等问题，建立了更清晰、更易维护的代码结构。

## 改进前后对比

### 1. 目录结构

**优化前：**
```
/opt/amaxtool/
├── check.sh              # 孤立脚本
├── col.sh                # 颜色函数（多处复制）
├── command               # 依赖列表
├── env.sh                # 环境变量
├── start                 # 主菜单
├── diskcheck/            # 子菜单脚本
├── getlog/               # 子菜单脚本
├── hwtest/               # 子菜单脚本
├── soft/                 # 子菜单脚本
├── sysinfo/              # 子菜单脚本
├── system/               # 子菜单脚本
├── tool/                 # 二进制工具
└── python/               # Python 脚本（散乱）
```

**优化后：**
```
/opt/amaxtool/
├── bin/
│   └── amax-tool         # 统一主入口
├── lib/                  # 公共库（去重）
│   ├── core.sh           # 核心函数
│   ├── env.sh            # 环境配置
│   └── utils.sh          # 工具函数
├── modules/              # 功能模块（标准化）
│   ├── check/
│   ├── diskcheck/
│   ├── getlog/
│   ├── hwtest/
│   ├── soft/
│   ├── sysinfo/
│   └── system/
├── tools/                # 二进制工具
├── README.md             # 使用文档
├── install.sh            # 安装脚本
└── （旧目录保留软链接兼容）
```

### 2. 代码改进

| 项目 | 优化前 | 优化后 |
|------|--------|--------|
| 代码重复 | col.sh 复制到 diskcheck/ | 统一使用 lib/core.sh |
| 路径引用 | `../env.sh` 相对路径 | `$SCRIPT_DIR/lib/env.sh` 绝对路径 |
| 错误处理 | 大多无错误检查 | 统一的错误处理和日志 |
| 颜色输出 | 各自实现 | 统一函数库 |
| 日志记录 | 无 | 完整的日志系统 |
| root 检查 | 部分脚本有 | 统一 check_root() 函数 |
| 代码风格 | 混用 tab/空格 | 统一使用空格，4 缩进 |
| 变量命名 | 大小写混乱 | 只读变量使用大写，局部用小写 |

### 3. 新增功能

- **统一日志系统**：所有操作记录到 `/var/log/amax-tool/amax-tool.log`
- **进度显示**：添加了进度条和旋转等待动画
- **确认提示**：危险操作前需要用户确认
- **语法检查**：安装时自动检查脚本语法
- **快捷方式**：安装后可直接使用 `amax-tool` 命令

### 4. 修复的问题

1. **路径问题**：原脚本使用 `../env.sh` 等相对路径，在某些情况下会失效
2. **权限检查**：许多脚本未检查是否以 root 运行
3. **命令缺失**：未检查依赖命令是否存在
4. **变量泄漏**：大量变量未使用 local 声明，可能污染全局命名空间
5. **Python shebang**：`#!/bin/python3` 改为 `#!/usr/bin/env python3`
6. **缺乏注释**：新代码添加了详细注释和文档

### 5. 向后兼容

为了保持兼容性：
- 原 `start` 脚本作为软链接指向新入口
- 各子目录的入口脚本（如 `sysinfo/run.sh`）作为软链接保留
- `col.sh` 和 `env.sh` 作为软链接保留
- 旧数据文件和日志保留在原位置

## 使用方式

### 新方式（推荐）

```bash
# 安装
sudo /opt/amaxtool/install.sh

# 运行
sudo amax-tool
```

### 兼容旧方式

```bash
# 仍然可用
sudo /opt/amaxtool/start
cd /opt/amaxtool/sysinfo && ./run.sh
cd /opt/amaxtool/system && ./menu.sh
```

## 性能改进

- 模块按需加载，减少资源占用
- 错误提前检测，避免无效操作
- 日志异步写入，不影响主流程

## 安全改进

- 所有敏感操作前要求确认
- 文件操作前自动备份
- root 权限检查前置
- 临时文件安全清理
